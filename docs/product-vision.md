# 产品愿景与功能草案

> 状态：草案，随讨论迭代。

## 定位

为指定主播提供类似 [laplace.live](https://laplace.live) 的 **粉丝入口站**：第一眼有记忆点，再分散到日程、内容、工具等能力。

## 首页：欢迎图 + 可点击气泡

- **欢迎图**：可用约稿立绘，或先用 HTML/CSS 占位，后续替换。
- **气泡导航**：多个浮层/气泡按钮，跳转到不同功能页（日程、歌切、工具、公告等）。
- **优点**：视觉统一、扩展新入口时只增加气泡与路由，不必每次改整站导航结构。

## 「日志 / 博客」指什么

在 Laplace 类站点里，通常指 **运营方或站长发布的图文**：开发记录、重大公告、随笔等，按时间线展示；与 **B 站动态** 不同，内容完全由本站编辑，适合长文与固定入口。  
本站是否采用、是否公开投稿，可后续定夺。

## 与直播强相关的核心能力（目标）

1. **录播日程表**  
   - 自动抓取该主播在 B 站 **上传的直播回放**，整理为 **日程 / 列表**（字段：标题、时间、链接、封面等）。  
   - 实现上需考虑 B 站接口与合规、抓取频率、缓存与失败重试。

2. **歌切 / 音频区**  
   - **管理员** 上传或维护主播相关 **唱歌音频**（或外链），前台展示播放列表。  
   - 需权限、存储、版权与审核流程（仅管理员可操作）。

## 直播小工具

- 例如：**OBS 用虚拟键盘** 等独立工具。  
- 形态可以是：本站「工具」页只做 **说明 + 链接** 到独立部署地址，或 **嵌入** / **同域子路径** 部署（与备案、HTTPS 策略一致即可）。

**已有工程（另仓，进行中）**：[`TimeZone-Virtual-Keyboard` @ `feat/agent-packaging-and-oneclick-start`](https://github.com/jiaxuli960429-dotcom/TimeZone-Virtual-Keyboard/tree/feat/agent-packaging-and-oneclick-start)（云端后端、本地 Agent、叠加页与打包脚本等，约一半进度）。汇总见 [related-repositories.md](related-repositories.md)。

## 非目标（现阶段）

- 不承诺与 Laplace 功能一一对齐；以主播与维护者真实需求为准。
